# الأمان — Souqna Security

## المصادقة والجلسات
- **كلمات المرور:** Argon2id (argon2-cffi) مع ترقية شفافة للمعاملات عند الدخول
  (`needs_rehash`). لا تُخزَّن كلمة مرور نصية أبداً.
- **JWT:** HS256 بمفتاح `SECRET_KEY` (ولّده بـ `openssl rand -hex 32`). Access قصير
  العمر (30 دقيقة) + Refresh (30 يوماً) مع تدوير عند كل تحديث، والنوع مثبَّت داخل
  الحمولة فلا يُقبل Refresh مكان Access.
- **الويب:** نفس الـ JWT في كوكيز `HttpOnly; SameSite=Lax` (+ `Secure` عبر
  `COOKIE_SECURE=true` خلف HTTPS). لا يصل JavaScript إلى التوكنات.
- **CSRF:** نمط double-submit — كوكي `sq_csrf` عشوائي + حقل مخفي في كل نموذج POST،
  مقارنة بزمن ثابت (`secrets.compare_digest`).
- الحساب المحظور/المعطَّل يُرفض في كل مسارات التحقق (API وويب) حتى لو كان توكنه صالحاً.

## المدخلات والرفع
- تحقق شامل عبر Pydantic (أطوال، أنماط، نطاقات؛ رقم الهاتف يُطبَّع ويُتحقق منه).
- **الصور:** حد الحجم `MAX_UPLOAD_MB` ⇠ فتح وتحقق عبر Pillow ⇠ **إعادة ترميز JPEG**
  (يُبطل الملفات المموّهة والحمولات المدفونة ويمسح EXIF/الموقع الجغرافي) ⇠ تصغير حتى
  1600px. أسماء الملفات UUID — لا مدخلات مستخدم في المسارات.
- القوالب (Jinja2) تُهرَّب تلقائياً — لا حقن HTML من محتوى المستخدمين.
- الاستعلامات كلها عبر SQLAlchemy بمعاملات مربوطة — لا حقن SQL.

## حدود الطلبات (Rate Limiting)
- slowapi حسب IP الحقيقي (uvicorn `--proxy-headers` خلف nginx).
- افتراضي `100/minute` وللمصادقة والبلاغات `10/minute` (قابلة للضبط بالبيئة).
- عبر `REDIS_URL` تصبح العدادات مشتركة بين عدة workers/خوادم.

## الصلاحيات
- دوران: `user` و`admin`. مسارات `/api/admin` و`/admin` تتحقق من الدور بعد المصادقة.
- ملكية الموارد مفروضة في الخادم (تعديل/حذف إعلان، قراءة محادثة، إشعارات).
- **سجل تدقيق** غير قابل للتعديل لكل إجراء إداري (من، ماذا، على من، متى) — يشمل
  إجراءات النظام الآلية (الإخفاء التلقائي).

## الإشراف على المحتوى
- إبلاغ مصنّف على الإعلانات والمستخدمين، بلاغ مفتوح واحد لكل (مبلّغ، هدف).
- إخفاء تلقائي بعد `REPORTS_AUTO_HIDE_THRESHOLD` بلاغات مفتوحة بانتظار المراجعة.
- إجراءات المشرف: إخفاء/حذف إعلان، حظر مستخدم، رفض البلاغ — مع إشعار المتأثر.

## البنية والتشغيل
- الحاوية تعمل **بمستخدم غير جذري** (appuser) ولا تكشف سوى منفذ داخلي خلف nginx.
- الأسرار كلها في `.env` (خارج git — انظر `.gitignore`)؛ `SECRET_KEY` وكلمات مرور
  القاعدة والمدير **يجب** تغييرها قبل النشر.
- Postgres غير منشور على الشبكة (بدون ports في compose).
- nginx يحدّ حجم الجسم (`client_max_body_size`) فوق حد الرفع بقليل.
- سجلّات منظّمة إلى stdout (صديقة لـ Docker) دون بيانات حساسة.
- نسخ احتياطي مجدول + سياسة احتفاظ (`backend/scripts/backup.sh`).
- تيليغرام اختياري تماماً؛ فشله لا يؤثر على أي طلب (fire-and-forget مع تسجيل).

## المتبقي مستقبلاً (انظر ROADMAP)
توثيق OTP بالهاتف (F2)، قائمة إبطال للتوكنات المسروقة، ترويسات CSP، 2FA للمشرفين.
