# خطة الاختبار — Souqna Test Plan

طبقتان، وكلاهما **يعملان فعلاً في المستودع** (ليسا خطة نظرية):

## 1) اختبارات الخلفية (pytest) — `backend/tests/`
سريعة (ثوانٍ)، تعمل بلا Docker عبر SQLite + عميل ASGI، وتغطي الـ API والويب معاً.

```bash
cd backend
pip install -r requirements-dev.txt
pytest            # 34 اختباراً
```

| الملف | يغطي |
|---|---|
| test_security.py | Argon2id، أنواع التوكنات، تطبيع رقم الهاتف |
| test_auth_api.py | تسجيل/تكرار/دخول خاطئ/تحديث/me/رفض بلا توكن |
| test_listings_api.py | CRUD، رفع صورة حقيقية ورفض ملف مزيّف، عدّاد المشاهدات، ملكية التعديل، البحث والفلاتر والفرز، إخفاء الإعلان عن غير مالكه |
| test_social_api.py | مفضلة (+إشعار المالك)، محادثة كاملة بالاتجاهين (+علامات القراءة وحجب الغرباء)، مسارات الإشعارات |
| test_moderation_api.py | تحقق البلاغات، منع التكرار، **الإخفاء التلقائي عند العتبة**، معالجة/رفض المشرف، الحظر يمنع الدخول، سجل التدقيق |
| test_admin_api.py | الإحصاءات، حظر/أدوار، إخفاء/حذف إداري (+إشعار المالك)، أقسام، منع غير المدير (403) |
| test_web.py | كوكيز الدخول، **رفض CSRF المزوّر**، إعادة توجيه الصفحات المحمية، تدفق سوق كامل عبر نماذج HTML (نشر بصورة ⇠ مفضلة ⇠ محادثة ⇠ بلاغ ⇠ تعديل ⇠ بيع)، صلاحيات `/admin` |

بيئة الاختبار تخفض عتبة الإخفاء التلقائي إلى 3 (env في conftest) لتغطيتها بسرعة.

## 2) فحص المتصفح الشامل (Playwright) — `e2e/`
Chromium حقيقي ضد **المكدّس الكامل** (nginx + api + Postgres) — يفحص ما يراه المستخدم.

```bash
docker compose up -d --build
pip install playwright pytest && playwright install chromium
E2E_ADMIN_PASSWORD=<من .env> python -m pytest e2e/ -v    # 16 تدفقاً
```
متغيرات: `E2E_BASE_URL` (افتراضي http://localhost:8080)، `E2E_ADMIN_PHONE/PASSWORD`،
و`E2E_CHROMIUM` لتمرير مسار Chromium مثبّت مسبقاً.

التدفقات المفحوصة (كل واحدة اختبار مستقل):
1. الرئيسية: أقسام + بحث من الشريط العلوي.
2. تسجيل حسابين (بائع/مشترٍ) عبر النماذج.
3. دخول خاطئ يظهر خطأ عربياً ثم دخول صحيح.
4. نشر إعلان بصورة فعلية والتحقق من صفحة التفاصيل.
5. البحث يجده؛ فلتر السعر يستبعده؛ فلتر المحافظة يعيده.
6. المشتري يضيفه للمفضلة ويراه في صفحتها.
7. محادثة كاملة بالاتجاهين (مشترٍ ⇠ بائع ⇠ مشترٍ).
8. إشعارات البائع (مفضلة+رسالة) ثم «تحديد الكل كمقروء».
9. الإبلاغ عن الإعلان من صفحة التفاصيل.
10. المالك يعدّل السعر ويحوّل الحالة (مباع ⇠ نشط).
11. المدير: الإحصاءات ثم معالجة البلاغ بإخفاء الإعلان + سجل التدقيق.
12. المخفي لا يظهر في البحث؛ المدير يعيد إظهاره فيعود.
13. حظر المشتري يمنع دخوله؛ رفع الحظر يعيده.
14. صفحة إدارة الأقسام.
15. تسجيل الخروج يعيد واجهة الزائر.
16. `/api/health` و`/api/docs` عبر nginx.

**آخر تشغيل موثّق: 34/34 pytest و16/16 Playwright — الكل أخضر.**

## متى تعيد التشغيل؟
- pytest بعد كل تعديل خلفية (ثوانٍ).
- Playwright قبل أي نشر أو ترحيل (دقائق) وبعده مباشرة على البيئة الجديدة عبر
  `E2E_BASE_URL=https://…` كفحص قبول.
